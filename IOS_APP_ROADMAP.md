## 1. Обзор архитектуры

- **Клиент**: нативное SwiftUI-приложение, минимум **iOS 26** (устройства
  iPhone 11+). Отдельный репозиторий **`Gennety-iOS`** (Xcode + SPM).
- **Бэкенд**: без изменений топологии — iOS-клиент является потребителем
  существующего `https://dating-api.gennety.com/v1/*` (JWT + refresh-ротация).
  Один дроплет, одна база, **общий пул матчинга** (`User.platform ∈
  {telegram, mobile, both}`, mobile-only — отрицательный `telegramId`).
- **Модель данных/реалтайм**: поллинг активных экранов (как в Mini Apps,
  ~4 с) + APNs-события. WebSocket/SSE в v1 нет; ощущение «живого AI» даёт
  клиентская typing/шиммер-анимация.
- **Продуктовые инварианты неприкосновенны**: blind decision, отсутствие
  user-to-user чата (кроме тайм-боксированного прокси-окна), обязательная
  верификация, отсутствие like/dislike-механик, недельный ритм дропа.

```
┌─────────────┐   JWT /v1/*    ┌──────────────────────────┐
│ Gennety-iOS │ ─────────────▶ │ apps/bot (Node, дроплет) │
│  (SwiftUI)  │ ◀───────────── │  Express :3101 + workers │
└─────┬───────┘   APNs (.p8)   └────────┬─────────────────┘
      │ StoreKit 2                      │ Postgres+pgvector (Supabase)
      │ Persona iOS SDK                 │ OpenAI / Rekognition / Places
      │ Twilio Verify (через /v1)       │ Resend / Persona / Twilio
```

## 2. Инфраструктура и репозиторий

| Решение | Значение |
|---|---|
| Репозиторий | Отдельный `Gennety-iOS` (прецедент: `gennety-dating-dashboard`) |
| Дроплет | Новый НЕ создаётся; iOS — клиент существующего API |
| Deployment target | iOS 26 (нативный Liquid Glass без фолбэков) |
| Dynamic Island | Аппаратный бонус iPhone 14 Pro+ / всех 15/16/17; на остальных Live Activity живёт на локскрине — функциональность идентична |
| Apple Developer | Платный аккаунт есть; нужен APNs Auth Key `.p8` (token-based) |
| Масштаб | ~10 тыс. через 3 мес., ~50 тыс. через 6 мес. |

**Зачистка Expo-наследия (бэкенд)**: удалить `mobile-handoff/`,
`EXPO_ACCESS_TOKEN`, Expo-транспорт в `services/push.ts` (замена — прямой
APNs). **Остаются**: весь `/v1/*`, `User.platform`, `pushToken/pushPlatform`
(значение `apns`), Aether (`/v1/chat/*`) — он становится основой iOS-чата.

**⚠️ Подготовка к масштабу (отдельный этап roadmap, не сейчас)**: один
Node-процесс (бот + API + 14 cron) на одном дроплете к 50 тыс. станет узким
местом. Минимальный план: вертикальный апгрейд дроплета → вынос public API в
отдельный PM2-процесс → при горизонтальном масштабировании перенос in-memory
rate-limiter'ов в Redis/Postgres. Нагрузка пиковая и предсказуемая
(четверговый батч), поэтому этап идёт после MVP, по метрикам.

## 3. Интеграция сервисов

### 3.1 Аутентификация и регистрация

- **Студенческий трек**: OTP на университетскую почту — существующие
  `/v1/auth/otp/request|verify` работают как есть.
- **Общий трек**: телефонный рельс Telegram (`message.contact`) на iOS
  невозможен → **Twilio Verify** (SMS OTP). Новые эндпоинты
  `/v1/auth/phone/request|verify`. Вход в существующий phone-track аккаунт —
  SMS на сохранённый `User.phone` (`@unique` защищает от дублей).
- **Sign in with Apple**: НЕ в MVP (не заменяет ни один рельс верификации).
- Бюджетная пометка: ~$0.05/верификация Twilio × десятки тысяч — в юнит-экономику.

### 3.2 Контракт API и версионирование

- **OpenAPI-спека** для `/v1/*` живёт в бэкенд-репо; Swift-клиент генерируется
  `swift-openapi-generator` — два репо не разъезжаются.
- Новый **`GET /v1/app/config`**: feature-флаги клиента + минимальная
  поддерживаемая версия (kill-switch принудительного обновления — в App Store
  нет мгновенного отката клиента).
- Изменения API — только аддитивные внутри `/v1`.

### 3.3 Push-инфраструктура

- Прямой **APNs** (token-based `.p8`) вместо Expo — обязательное условие
  Live Activities.
- `POST /v1/me/push-token` расширяется: `pushPlatform=apns` + регистрация
  **Live Activity push-token'ов** (start/update tokens) для сценария
  «День свидания».

### 3.4 Платежи (Date Tickets)

- **С первого релиза.** **StoreKit 2** + серверная верификация транзакций
  (App Store Server API + Server Notifications V2).
- Третий режим `appstore` в `services/ticket-payment.ts` рядом с mock/Stars.
- **Общий кошелёк** легален (guideline 3.1.2): тикеты из Telegram видны и
  тратятся на iOS, поскольку те же тикеты покупаемы через IAP in-app.
- Famine-скидка на iOS = отдельный IAP-продукт (спроектировать на реализации).
- IAP-продукты: consumable `ticket_1`, `ticket_3`, `ticket_6` (+ discount SKU).

### 3.5 Верификация (Persona)

- Нативный **Persona Inquiry SDK для iOS** вместо hosted-URL: та же серверная
  труба (`verification-pipeline`), HMAC-webhook остаётся единственным путём к
  `verified`. Небольшое расширение `/v1/verification/*` под нативный клиент.

### 3.6 AI-чат (Aether)

- `/v1/chat/*` реанимируется как серверная основа чат-экрана.
- **Расширение контракта — `ui_hint`**: типизированное поле в ответе чата и
  онбординг-интервью, указывающее клиенту, какой нативный инлайн-контрол
  отрендерить (age_wheel / date_picker / map / photo_upload / choice_chips…).
- **Решение по матчу**: новый эндпоинт классификации текстового ответа,
  переиспользующий логику `handlers/matching/decision-text.ts` (keyword
  fast-path + LLM fallback). Текст никогда не коммитит решение — только
  явный тап по подтверждающей карточке.

### 3.7 Сводка новых/изменяемых серверных работ

| Работа | Тип |
|---|---|
| APNs-транспорт в `services/push.ts` + Live Activity tokens | замена |
| `/v1/auth/phone/*` (Twilio Verify) | новое |
| `GET /v1/app/config` | новое |
| OpenAPI-спека `/v1/*` | новое |
| `ui_hint` в `/v1/chat` и `/v1/onboarding/interview*` | расширение |
| Эндпоинт классификации decision-текста | новое |
| `appstore`-режим ticket-payment + App Store Server API | новое |
| Пауза/возобновление поиска (`PATCH /v1/me/status`) | новое |
| Freeze-аккаунта для mobile (сейчас Telegram-only) | расширение |
| Persona native SDK init/event | расширение |
| Зачистка Expo | удаление |

## 4. Нативные iOS-фичи

### 4.1 Live Activities + Dynamic Island (обе в MVP)

1. **«Решение по матчу»** — 24-часовое окно питча. Обратный отсчёт дедлайна
   через `Text(timerInterval:)` (без пушей — лимит 8ч обновлений не мешает).
   Island compact: таймер + силуэт; expanded: «Ваш AI нашёл вам пару ·
   осталось Xч».
2. **«День свидания»** — старт в **T-5ч** (синхронно с айсбрейкерами),
   до T+2ч (7ч жизни, в лимитах 8/12ч). Стадии: айсбрейкеры пришли →
   wingman-подсказка (T-1.5ч) → анонимный чат открыт (T-30м) → отсчёт до
   встречи → «как прошло?». Кнопки (App Intents): Открыть в Maps / Войти в
   чат. Обновления стадий — через APNs Live Activity push.

### 4.2 Виджет

Один умный **«Gennety Status»** (home screen, read-only + deep-link):
отсчёт до четверга 18:00 ↔ «💫 Свидание через 2д 4ч · Venue» ↔
пауза/famine — зеркало логики пинового баннера Telegram. Данные:
`/v1/countdown` + текущий матч. Без Accept/Decline (философия осознанности).

### 4.3 Уведомления

- **Пре-пермишн экран**: красивый брендовый экран ДО системного диалога.
  Момент показа — сразу после активации: «Ваш AI начал поиск — узнайте
  первым, когда он кого-то найдёт» + макет будущего уведомления. Тап
  «Согласен» → системный `Allow Notifications`. Отказ — повторное
  предложение в контекстные моменты (не чаще раза в цикл).
- **Карта уведомлений** (без надоедания; тихие часы уже соблюдает сервер):

| Событие | Пуш | Особенности |
|---|---|---|
| Ср 18:00 — pre-match teaser | ✅ | Предвкушение: «завтра ваш AI покажет находку» |
| Чт 18:00 — питч | ✅ rich | **Блюр-фото партнёра** («разблокируй, чтобы увидеть») через Notification Service Extension; действие — только «Посмотреть» |
| Чт 18:15 — no-match | ✅ | Эмпатичный тон, famine-скидка при tier ≥ 2 |
| Nudge решения (3ч/10ч) | ✅ | Обычный приоритет |
| Nudge планирования (6ч/12ч) | ✅ | Обычный приоритет |
| «Скоро свидание» | ❌ | Закрывается Live Activity — пуш не нужен |
| T-1.5ч safety-брифинг (жен.) | ✅ | **time-sensitive** |
| T-30м «чат открыт» | ✅ | **time-sensitive**; **inline-ответ** прямо из уведомления → прокси-чат |
| T+24ч фидбек | ✅ | Быстрые кнопки «🔥 / 😐» + «Рассказать подробнее» |
| Re-engagement онбординга | ✅ | Существующая 5-шаговая цепочка |

### 4.4 Apple Wallet — этап 2

Пасс на запланированное свидание (venue, адрес, время, имя партнёра;
relevance по времени/гео). Отложен: частично дублирует Live Activity
«День свидания».

### 4.5 Хаптика

Язык вибраций — часть дизайн-системы: плавная, **нарастающая** вибрация в
жесте поиска/паузы; деликатные отклики на ключевых коммитах (решение,
оплата, подтверждение свидания).

## 5. Дизайн-система

- **Этос**: luxury dating, Pinterest-эстетика, «дорогие» плавные анимации.
  Приложение короткой сессии — человек не «залипает», поэтому каждый экран
  обязан быть безупречным; витринные — «Сегодня» и «Чат».
- **Liquid Glass — нативный** (`glassEffect`, `GlassEffectContainer`,
  iOS 26). Системное поведение стекла при скролле/свете не имитируем — берём
  из коробки.
- **Бренд в стекле**: бургунди `#8B253B` — только акценты (CTA, последнее
  слово заголовка, таймеры, свечения — экономно). Заголовки — Archivo Black.
  Тёмная база `#030303` / светлая `#F5F5F5`; плёночное зерно — только на
  тёмной. Слоган-голос бренда сохраняется.
- **Темы**: по умолчанию — системная тема iPhone; переключатель в настройках;
  выбор синхронизируется в `User.theme` (серверные PNG-карточки Telegram
  совпадают с клиентом).
- **Сцена за стеклом** (гипотеза, тестируем): ambient-слой — медленное
  бургунди-свечение на почти-чёрном; фото партнёра за стеклом на экранах
  матча; инлайн-контролы чата — безрамочные стеклянные панели. **Фон —
  отдельный сменный слой** (design-токен/флаг): один переключатель
  откатывает до стандартного плоского фона.
- **Typing/шиммер-анимация** — фирменный паттерн «AI думает/печатает»
  (замена серверного стриминга в v1).
- **Инлайн-контролы гибридного чата** — единая библиотека: age-wheel,
  календарь, карта, фото-зум, choice-chips. Стиль один во всех флоу.

## 6. Экраны и флоу

### 6.1 Онбординг (8 шагов)

Визуальное интро → согласие/ToS → развилка треков (почта-OTP / Twilio-SMS) →
город → AI-memory выбор (Magic Prompt: deep-link «Открыть ChatGPT/Claude» +
поле вставки / отказ → fallback-анализ) → **гибридный чат-интервью**
(серверный коллектор владеет порядком; `ui_hint`-контролы) → фото (мин. 4,
нативный пикер, серверная валидация как в Telegram) → Persona SDK →
активация → **пре-пермишн уведомлений**. Язык — из системной локали
(en/ru/uk/de/pl, fallback en), экрана темы нет (следуем системе).

### 6.2 Таб «Сегодня» — статус-сцена

- Сверху — **полукруглое окно-коллаж**: медленно прокручивающиеся сплошь
  замыленные «профили». Реализация: заготовленные художественные ассеты
  (не реальные пользователи — API раздачи чужих фото не существует и не
  создаётся), поверх — **реальный счётчик** («2 340 в поиске в Киеве»).
- Снизу — фотография пользователя. **Жест**: поднял свою фотографию к
  коллажу (нарастающая плавная вибрация) → активный поиск; вытащил → пауза.
  Статус — одна ненавязчивая строка. Защита от случайности: haptic +
  «Отменить»-тост.
- Состояния: отсчёт до дропа / активный матч (вход в питч) / предстоящее
  свидание (вход в хаб) / famine (эмпатия + discount-CTA) / пауза.

### 6.3 Кино-питч (решение по матчу)

Три акта, полноэкранно, без like/dislike:
1. **Появление** — фото партнёра, Ken Burns, имя+возраст (Archivo Black,
   последнее слово бургунди).
2. **Голос AI** — питч печатается шиммером; текст «отвоёвывает» нижнюю
   треть — растущая стеклянная панель замыливает фото под собой;
   разворачивается свайпом. Synergy — деликатная дуга/свечение (70–99).
3. **Решение разговором** — AI спрашивает; человек отвечает своими словами;
   сервер классифицирует интент; из ответа «вытекает» карточка: «💫 Иду»
   или guard-подтверждение отказа красным. Blind-инвариант нетронут.
Дедлайн — тонкий таймер сверху (синхронно с Live Activity). Rematch (будущее)
ложится без переделки: каждый профиль — своя полноэкранная история.

### 6.4 После взаимного «да»

- **Тикет-гейт**: использовать из кошелька / купить (StoreKit); мужской
  сценарий «оплатить за обоих»; экран-сюрприз «{имя} уже оплатил ❤️».
- **Календарь**: нативная сетка слотов (6 дат × 6 времён) с живыми отметками
  партнёра (поллинг), 4 состояния слота, initiator-offers/responder-decides.
- **Точка отправления + вайб**: нативная карта (поиск/тап/драг) + чат-ввод
  вайба (location-first порядок сохранён).
- **Подтверждение свидания**: нативный рендер date card (живой SwiftUI, не
  PNG) — venue, адрес, блюрб, add-to-calendar (EventKit).

### 6.5 День свидания и после

Хаб свидания (Maps, share с блюром, отмена с emergency-guard, report) →
Live Activity → айсбрейкеры → wingman-подсказка → **анонимный прокси-чат**
(T-30м…T+2ч, text-only, Report на каждом сообщении) → T+24ч форма фидбека
(слайдер химии 1–10, «второе свидание?», текст/голос).

### 6.6 Остальные табы

- **Чат** — Aether: гибридный AI-чат, история, фото-вложения.
- **Билеты** — кошелёк + магазин (IAP 1/3/6), история.
- **Профиль** — просмотр/редактирование (не-identity поля), фото,
  настройки (язык, тема, пауза), верификация, **freeze/delete** (freeze
  переносится на iOS), legal, report/help.

### 6.7 MVP-срез (этап 2 — проектируем после MVP)

Venue-change board · profile video · координация A/B (обмен контактами;
остаётся только прокси-чат C) · Apple Wallet · rematch · Sign in with Apple ·
Android (Kotlin/Compose — отдельная разработка).

### Риски и внешние зависимости

- **App Store review**: дейтинг = 17+, требования к UGC-модерации (закрыты:
  reports/strikes/moderation), обязательное удаление аккаунта (есть),
